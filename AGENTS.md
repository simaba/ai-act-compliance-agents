# Agent Instructions

This repo supports structured drafting and traceability for automotive AI compliance.

## Principles

1. Separate regulation text, interpretation, and project assumptions.
2. Keep legal, safety, and product claims traceable.
3. Flag uncertainty explicitly.
4. Prefer evidence-backed controls to generic assurances.
5. Always note where formal legal, safety, or certification review is still required.

## Delegation

- `risk-classifier` for AI Act risk categorization
- `conformity-checker` for checklist-based gap reviews
- `safety-mapper` for mapping hazards, misuse, and assurance arguments
- `oversight-designer` for human oversight mechanisms
- `post-market-monitor` for monitoring and feedback-loop plans
