# AI Act Compliance Agents v0.1.0

**Publication status:** Draft release notes. Complete the public-release checklist before creating a GitHub Release.

## Highlights

- Structured, fictional traceability-register inputs.
- Validation for stable control IDs, source provenance, ownership, review state, and evidence pointers.
- JSON Schema, public input contract, and synthetic sample input.
- Practitioner methodology and source-provenance baseline.

## Intended use

Use this release to organize fictional or sanitized draft traceability artifacts and to discuss what a review-ready control register should contain.

## Important limits

- It does not determine legal applicability, compliance, conformity, safety, or production readiness.
- It does not validate evidence quality or source interpretation.
- It must not contain real employer, customer, supplier, vehicle, feature, safety, legal, security, privacy, or release information.
