# Public Release Checklist

Use this checklist before changing the repository visibility to public.

## Required before publishing

- [ ] Confirm all examples are fictional, synthetic, or fully sanitized.
- [ ] Confirm no real compliance positions, regulatory conclusions, or legal interpretations are committed.
- [ ] Confirm no real safety analyses, hazards, incidents, test results, or safety requirements are included.
- [ ] Confirm no employer, supplier, vendor, customer, or internal project names appear in examples.
- [ ] Confirm no unreleased product names, roadmaps, architecture, release gates, or internal approval paths are included.
- [ ] Confirm no API keys, tokens, credentials, connector details, account identifiers, or private paths are present.
- [ ] Confirm all EU AI Act, ISO 26262, and ISO 21448 references are framed as practitioner examples, not official determinations.
- [ ] Confirm the README does not imply legal advice, conformity assessment, certification, safety-case approval, regulatory approval, or endorsement.
- [ ] Confirm the license, README, and quick-start commands are accurate.

## Recommended before promotion

- [ ] Add one fictional traceability example.
- [ ] Add a methodology note for how mappings should be reviewed.
- [ ] Add source references for any regulatory or standards concepts.
- [ ] Add tests for edge cases in traceability utilities.
- [ ] Run tests locally after a fresh clone.

## Final manual review

Before publishing, search the repo for sensitive terms such as company names, vehicle program names, internal acronyms, supplier names, customer names, emails, tokens, legal conclusions, safety classifications, and private paths.

This checklist is a publication aid, not a security, legal, compliance, or safety guarantee.
